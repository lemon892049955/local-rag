## 一、产品架构

### 1.1 产品定位

BuddyKnow 是一个**本地化优先的个人知识库管理系统**，帮助用户将碎片化知识（公众号文章、小红书笔记、知乎回答等）转化为结构化的个人知识库，并提供 AI 驱动的智能问答服务。

### 1.3 功能矩阵

| 模块 | 功能 | 描述 |
|------|------|------|
| **内容入库** | URL 入库 | 支持公众号、小红书、知乎、B站等主流平台 |
| | 文件上传 | PDF、图片、音频自动解析入库 |
| | 批量入库 | 多 URL 并行处理，SSE 进度推送 |
| **知识检索** | 语义搜索 | 向量检索 + BM25 混合召回 |
| | RAG 问答 | LLM 基于知识库生成回答，标注来源 |
| | 流式输出 | 逐字显示，实时响应 |
| **Wiki 编译** | 自动编译 | 入库文章自动生成 Wiki 页面 |
| | 分类体系 | 动态分类 + 语义子分类 |
| | 知识图谱 | 标签关联可视化 |
| **微信机器人** | iLink Bot | 微信个人号接入，转发即入库 |
| | 智能对话 | 发文字搜索，发链接入库 |
| **发现模块** | 热榜抓取 | Hacker News、知乎、V2EX 等 |
| | 质量筛选 | 基于知识库分布的内容筛选 |
| | 一键入库 | 审核后快速入库 |
| **数据埋点** | 行为追踪 | 搜索、入库、点击、反馈 |
| | 统计分析 | 热门查询、无结果词、趋势分析 |

### 2.3 核心技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **Web 框架** | FastAPI + Uvicorn | 高性能异步 API |
| **前端** | Alpine.js + Tailwind CSS | 轻量级响应式 UI |
| **LLM** | DeepSeek / Kimi | OpenAI 兼容接口 |
| **向量数据库** | ChromaDB | 本地嵌入向量存储 |
| **Embedding** | sentence-transformers | BGE-small-zh 中文向量模型 |
| **关键词检索** | BM25 + jieba | 中文分词 + 倒排索引 |
| **重排序** | Cross-Encoder | BGE-reranker-base |
| **微信机器人** | wechat-ilink-bot | 微信个人号 SDK |
| **存储** | Markdown + SQLite | 文件 + 轻量数据库 |

### 2.4 数据流架构

#### 2.4.1 入库流程

```
URL/File → Fetcher → RawContent
                           │
                           ▼
                      LLM Cleaner
                           │
                           ▼
                      MarkdownEngine → data/*.md
                           │
                           ▼
                      VectorIndexer → vectordb/
                           │
                           ▼
                      WikiCompiler → wiki/topics/*.md
```

#### 2.4.2 检索流程

```
Query → Intent Detection
              │
              ▼
         Query Rewrite
              │
              ├──────────────┐
              ▼              ▼
         Vector Search   BM25 Search
              │              │
              └──────┬───────┘
                     ▼
               RRF Fusion
                     │
                     ▼
               Reranker
                     │
                     ▼
               Top-K Chunks
                     │
                     ▼
               LLM Generation
                     │
                     ▼
               Answer + Sources
```

## 三、部署架构

### 3.1 单机部署（推荐）

```bash
# 1. 安装依赖
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env 填入 API Key

# 3. 启动服务
uvicorn main:app --host 0.0.0.0 --port 8900
```

### 3.2 Docker 部署

```bash
docker-compose up -d
```

### 3.3 微信机器人配置

```bash
# 扫码登录
python ilink/login.py

# 机器人会随主服务自动启动
```

## 四、性能指标

| 指标 | 数值 | 说明 |
|------|------|------|
| 搜索响应 | < 2s | 含向量召回 + 重排序 + LLM 生成 |
| 入库耗时 | 3-10s | 含抓取 + 清洗 + 索引 + Wiki 编译 |
| 并发支持 | ~10 QPS | 单机 uvicorn |
| 向量维度 | 384 | BGE-small-zh |
| 索引切片 | ~500/chunk | 按 512 token 切分 |

## 五、扩展性设计

### 5.1 水平扩展

- **向量数据库**：ChromaDB → Qdrant / Milvus（支持分布式）
- **存储**：本地文件 → S3 / MinIO
- **缓存**：内存会话 → Redis

### 5.2 多租户支持

- 数据隔离：`data/{user_id}/` 目录结构
- 向量索引：`collection_{user_id}` 分离
- API 认证：Token 绑定用户身份
