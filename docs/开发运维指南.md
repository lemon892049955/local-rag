| 服务器 (腾讯云) | 生产环境、用户使用 | http://124.222.99.141:8080 |
| GitHub | 代码托管 | https://github.com/lemon892049955/local-rag |

## 二、日常开发流程

### 1. 修改代码并测试

```bash
# 本地启动服务
cd /Users/teng/local-rag
source .venv/bin/activate
uvicorn main:app --host 0.0.0.0 --port 8900

# 浏览器访问 http://localhost:8900 测试
```

### 2. 确认无误后提交

```bash
git add .
git commit -m "描述你的修改"
git push origin main
```

### 3. 服务器更新部署

```bash
# SSH 登录服务器
ssh ubuntu@124.222.99.141
sudo -i

# 更新代码
cd /root/local-rag-new
git pull

# 重新构建并部署
docker stop local-rag
docker rm local-rag
docker build -t local-rag:v2.4 .
docker run -d --name local-rag -p 8080:8900 --restart always \
  -v /root/local-rag-new/data:/app/data \
  -v /root/local-rag-new/vectordb:/app/vectordb \
  -v /root/local-rag-new/wiki:/app/wiki \
  local-rag:v2.4
```

## 三、数据同步流程

### 数据位置说明

| 数据 | 本地路径 | 服务器路径 | 说明 |
|------|----------|------------|------|
| 原始文章 | `data/` | `/root/local-rag-new/data/` | Markdown 文件 |
| 向量索引 | `vectordb/` | `/root/local-rag-new/vectordb/` | ChromaDB |
| Wiki 页面 | `wiki/` | `/root/local-rag-new/wiki/` | 编译后的知识网络 |

### 本地 → 服务器（上传数据）

```bash
# 在 Mac 终端执行
# 先在服务器上复制数据到可访问位置
ssh ubuntu@124.222.99.141 "sudo cp -r /root/local-rag-new/data /home/ubuntu/ && sudo chown -R ubuntu:ubuntu /home/ubuntu/data"

# 从 Mac 上传
scp -r /Users/teng/local-rag/data ubuntu@124.222.99.141:/home/ubuntu/
scp -r /Users/teng/local-rag/vectordb ubuntu@124.222.99.141:/home/ubuntu/
scp -r /Users/teng/local-rag/wiki ubuntu@124.222.99.141:/home/ubuntu/

# 在服务器上移动到正确位置
ssh ubuntu@124.222.99.141 "sudo cp -r /home/ubuntu/data /root/local-rag-new/ && sudo cp -r /home/ubuntu/vectordb /root/local-rag-new/ && sudo cp -r /home/ubuntu/wiki /root/local-rag-new/"
```

### 服务器 → 本地（下载数据）

```bash
# 在服务器上准备数据
ssh ubuntu@124.222.99.141
sudo cp -r /root/local-rag-new/data /home/ubuntu/
sudo cp -r /root/local-rag-new/vectordb /home/ubuntu/
sudo cp -r /root/local-rag-new/wiki /home/ubuntu/
sudo chown -R ubuntu:ubuntu /home/ubuntu/data /home/ubuntu/vectordb /home/ubuntu/wiki
exit

# 在 Mac 下载
cd /Users/teng/local-rag
scp -r ubuntu@124.222.99.141:/home/ubuntu/data .
scp -r ubuntu@124.222.99.141:/home/ubuntu/vectordb .
scp -r ubuntu@124.222.99.141:/home/ubuntu/wiki .
```

## 四、常见场景

### 场景 1：修改前端 UI

1. 本地修改 `web/index.html`
2. 刷新浏览器测试
3. 满意后 `git push`
4. 服务器 `git pull` + 重启容器

### 场景 2：修改检索算法

1. 本地修改 `retrieval/` 下代码
2. 本地测试搜索效果
3. 满意后 `git push`
4. 服务器更新代码
5. **可选**：执行 `/reindex` 重建索引

### 场景 3：修改入库逻辑（需要重新入库）

**推荐：在服务器上操作**

1. 推送代码到服务器
2. 服务器上执行重新入库
3. 或者清空 `data/` 和 `vectordb/`，重新入库

```bash
# 服务器上重建索引
curl -X POST http://localhost:8900/reindex
```
