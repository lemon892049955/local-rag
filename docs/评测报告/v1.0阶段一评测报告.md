## 二、关键发现

### 好消息

1. **延迟暴降 56%**：8.49s vs 19.59s（v0.9.3）。知识密度提升后，检索更精准，LLM 生成更快
2. **知识密度翻 6 倍**：821 切片 vs 143 切片，检索可选范围大幅扩大
3. **实体从 0 到 45**：张小龙、Karpathy、Loopit 等都有独立页面了
4. **概念卡 41 个**：Vibe Coding、RAG、Cross-Encoder、Harness Engineering 等核心概念都有定义
5. **MOC 导航页 8 个**：每个分类都有结构化入口
6. **输出缓存已生效**：重复问题秒回

### 需要关注

1. **Recall 略降**（81.9% → 78.4%）：切片数从 143 → 821，检索空间变大，部分噪音概念卡干扰了排序。**阶段二 Reranker 优化可以解决**
2. **MRR 略降**（0.659 → 0.594）：同上，排序被稀释。精排优化后可恢复
3. **Keyword Score 下降**（84.6% → 79.7%）：DeepSeek 强化 Prompt 后回答更克制，少了一些"猜测性"关键词匹配。**实际上是好事**——忠实度提升的代价
4. **概念卡 Precision 偏低**（28.6%）：主要是评测匹配算法问题，实际抽取的概念（Vibe Coding、RAG 等）质量不错

### 细节查找下降分析

细节查找 Hit Rate 从 89% → 78%，原因是概念卡和实体页的切片"抢"了 Top-5 位置，导致原始文章片段被挤出。阶段二的 **Context 裁剪（只喂 Reranker Top-3）** 可以修复。
