> 个人碎片知识落库系统 — v0.9 (2026-04-10)

## 一、系统总览

```
┌─────────────── 入口层 ──────────────┐    ┌──── 处理层 ────┐    ┌──── 存储层 ────┐    ┌──── 智能层 ────┐
│                                     │    │                │    │                │    │                │
│  企微消息 ──┐                        │    │   LLM 清洗     │    │  Markdown      │    │  Wiki 编译     │
│  Web UI  ──┼─→ 统一入库管线 ────────┼──→ │  (元数据提取)   │──→ │  文件落库       │──→ │  (语义去重     │
│  AI 助手 ──┤   (ingest_pipeline)   │    │                │    │                │    │   LLM 分类)    │
│  CLI     ──┘                        │    │   基础降噪     │    │  向量索引       │    │                │
│                                     │    │  (规则+OCR)    │    │  (ChromaDB)    │    │  Taxonomy      │
└─────────────────────────────────────┘    └────────────────┘    │                │    │  (语义分类树)   │
                                                                 │  BM25 索引     │    │                │
┌─────────────── 检索层 ──────────────┐                          │  (jieba+pickle) │    └────────────────┘
│                                     │                          └────────────────┘
│  查询改写 → 向量+BM25 并行召回      │
│  → RRF 融合 → Reranker 精排        │
│  → Wiki+Data 双路上下文 → LLM 生成  │
│                                     │
└─────────────────────────────────────┘
```

**技术栈**：FastAPI + ChromaDB + SentenceTransformers + OpenAI SDK + AlpineJS + D3.js

## 二、模块架构

### 2.1 数据入库管线 (`services/ingest_pipeline.py`)

```
URL/文件 → 去重检查 → 路由分发 → 专用解析器 → 图片OCR → LLM元数据提取 → Markdown落库 → 向量索引 → Wiki编译入队
                        │
                ┌───────┼───────┐
                ↓       ↓       ↓
           微信公众号  小红书    通用网页
           (SSR解析)  (MCP优先)  (Readability)
                        ↓
                    ┌────┼────┐
                    ↓    ↓    ↓
                   PDF  图片  音频
                (pymupdf)(Kimi Vision)(Whisper)
```

| 模块 | 文件 | 核心能力 |
|------|------|---------|
| 统一管线 | `services/ingest_pipeline.py` | 7 阶段 + SSE 进度回调 |
| URL 路由 | `ingestion/router.py` | 域名自动分发 |
| 微信抓取 | `ingestion/wechat.py` | 图文穿插 OCR（占位符策略） |
| 小红书抓取 | `ingestion/xiaohongshu.py` | MCP 协议 + HTTP 降级 |
| 通用抓取 | `ingestion/general.py` | Readability + 知乎 API 特殊处理 |
| 文件分发 | `ingestion/dispatcher.py` | PDF/图片/音频类型检测 |
| 图片 OCR | `ingestion/vision_ocr.py` | 图片分类 → 专用 Prompt（文字/图表/表格/代码/照片） |
| PDF 解析 | `ingestion/pdf_parser.py` | pymupdf 文本提取 + 扫描件 OCR 降级 |
| 音频转录 | `ingestion/audio_transcriber.py` | Whisper API + 大文件自动分段 |
| LLM 清洗 | `transform/llm_cleaner.py` | 只提取元数据，正文保留原文 + 基础降噪 |
| Markdown 落库 | `storage/markdown_engine.py` | YAML Front-matter + Markdown 格式 |

### 2.2 检索引擎 (`retrieval/`)

```
用户查询
  ↓
[查询改写] LLM 生成 2-3 个检索 query（语义+关键词互补）
  ↓
[并行召回]
  ├── Data 路: 向量检索(Top-5) + BM25(Top-5) → RRF 融合 → Top-3
  └── Wiki 路: 向量检索(Top-2) + BM25(Top-2) → RRF 融合 → Top-2
  ↓
[Cross-Encoder Reranker] 精排 Top-5
  ↓
[双层 Context 组装] Wiki(宏观) + Data(微观)
  ↓
[LLM 答案生成] RAG Prompt → 综合回答 + 来源标注
```

| 组件 | 文件 | 说明 |
|------|------|------|
| 向量索引 | `retrieval/indexer.py` | ChromaDB + BGE embedding（支持 query instruction） |
| BM25 索引 | `retrieval/bm25.py` | jieba 分词 + pickle 持久化 |
| 语义切片 | `retrieval/chunker.py` | 按标题层级切分 + 摘要切片（Summary Chunk） |
| 混合检索 | `retrieval/hybrid_searcher.py` | 查询改写 + 并行多路召回 + RRF + Reranker + RAG |
| 精排器 | `retrieval/reranker.py` | Cross-Encoder (`bge-reranker-v2-m3`) |
| 分词器 | `retrieval/tokenizer.py` | jieba + 自定义用户词典 |

**当前 Embedding 模型**：`BAAI/bge-small-zh-v1.5`（中文 512 维）

### 2.3 Wiki 编译引擎 (`wiki/`)

```
新文章入库
  ↓
[语义去重门控] Embedding 找相似 Wiki 页面 → 注入 Prompt
  ↓
[LLM 编译计划] 决定新建/更新/跳过（含 Taxonomy 分类上下文）
  ↓
[执行编译]
  ├── 新建页面 → LLM 生成内容 → Taxonomy 自动分类
  └── 追加洞察 → LLM 提取增量 → Embedding 去重校验
  ↓
[索引更新] 重建 _index.md + 增量向量索引
```

| 组件 | 文件 | 说明 |
|------|------|------|
| 编译引擎 | `wiki/compiler.py` | 核心：计划→新建→追加→交叉引用，Append-only |
| 编译队列 | `wiki/compile_queue.py` | asyncio.Queue 串行编译，避免并发冲突 |
| 页面存储 | `wiki/page_store.py` | CRUD + 洞察追加 + 来源管理 |
| 索引构建 | `wiki/index_builder.py` | _index.md 自动生成 + 编译日志 |
| 语义分类 | `wiki/taxonomy.py` | LLM 语义分类（9 个预定义分类 + 自演化） |
| 健康检查 | `wiki/inspector.py` | 孤立页面/缺失引用/过时页面检测 |

### 2.4 AI 助手 (`assistant/`)

```
用户输入
  ↓
[意图识别] URL → 入库 | 问号/疑问词 → 搜索 | "状态" → 统计 | 其他 → 闲聊
  ↓
[上下文构建] 按意图调用对应模块获取数据
  ↓
[流式对话] 多轮会话 + SSE 流式输出
```

| 组件 | 文件 | 说明 |
|------|------|------|
| 意图识别 | `assistant/intent.py` | 规则优先：URL→入库，疑问词→搜索 |
| 对话引擎 | `assistant/chat_engine.py` | 多轮会话 + 流式 SSE |
| API 路由 | `assistant/router.py` | /api/assistant/chat (SSE) + chat-sync |
| 定时推送 | `assistant/scheduler.py` | 周报/回顾/关联推荐/健康报告 |

### 2.5 企微集成 (`wecom/`)

| 组件 | 文件 | 说明 |
|------|------|------|
| 回调处理 | `wecom/callback.py` | 接收转发消息 → 意图分发 → 异步入库/搜索/对话 |
| 消息发送 | `wecom/sender.py` | access_token 缓存 + 文本消息/卡片消息 |
| 加解密 | `wecom/crypto.py` | AES 加解密 + 签名校验 |

### 2.6 前端 (`web/index.html`)

单 HTML 文件，1039 行，5 个页面：

| 页面 | 功能 |
|------|------|
| 知识图谱 | D3 力导向图 + 标签云 + 时间轴 + Wiki 健康检查 |
| 知识索引 | 分类浏览 + 标签筛选 |
| 入库 | URL 入库（SSE 进度） + 文件上传（PDF/图片/音频） |
| 搜索 | 自然语言问答 + 来源展示 |
| AI 助手 | 右下角悬浮窗，流式对话 + 意图识别 + 通知推送 |

### 2.7 评估体系 (`eval/`)

| 组件 | 文件 | 说明 |
|------|------|------|
| 基础评估 | `eval/evaluator.py` | Recall@5 / MRR / Hit Rate / Keyword Score |
| 多维评估 | `eval/deep_eval.py` | 忠实度 + 上下文相关性 + 答案相关性 + 延迟切片 |
| 测试集 | `eval/test_set.json` | 52 条用例，7 类（精确/模糊/否定/长文/跨源...） |

## 四、数据流向

```
外部内容（公众号/小红书/知乎/PDF/图片/音频）
         │
         ↓
    ┌─ ingest_pipeline ─────────────────────────────────┐
    │  去重 → 抓取 → OCR → LLM清洗 → Markdown落库       │
    └───────────┬───────────────────────────────────────┘
                │
    ┌───────────┼───────────────────┐
    ↓           ↓                   ↓
data/*.md   vectordb/           wiki/
(原始文档)  (ChromaDB)       ┌────┼────┐
                            topics/ entities/ insights/
                              │
                        _taxonomy.yaml (语义分类)
                        _index.md (自动索引)
                        _log.md (操作日志)
```

## 五、配置参数

| 参数 | 当前值 | 说明 |
|------|--------|------|
| LLM Provider | Kimi (moonshot-v1-8k) | 支持切换 DeepSeek/Ollama |
| Embedding | BAAI/bge-small-zh-v1.5 | 中文 512 维 |
| Reranker | BAAI/bge-reranker-v2-m3 | Cross-Encoder 精排 |
| 向量数据库 | ChromaDB (持久化) | cosine 距离 |
| BM25 分词 | jieba + 用户词典 | pickle 缓存 |
| 服务端口 | 8900 (容器内) / 8080 (外部) | Docker 映射 |

## 六、部署架构

```
服务器: 腾讯云轻量应用 (上海, lhins-3mnho28y, 124.222.99.141)

Docker 容器: local-rag:v0.9.1
  ├── uvicorn main:app --host 0.0.0.0 --port 8900
  ├── Wiki 编译 Worker (asyncio 后台任务)
  └── 推送调度器 (asyncio 后台任务)

Volume 挂载:
  /root/local-rag-data       → /app/data       (原始文档)
  /root/local-rag-vectordb   → /app/vectordb   (ChromaDB)
  /root/local-rag-wiki       → /app/wiki/topics
  /root/local-rag-wiki-entities → /app/wiki/entities
  /root/local-rag-wiki-insights → /app/wiki/insights

端口:
  8080 (公网) → 8900 (容器) — Web UI + API
```
